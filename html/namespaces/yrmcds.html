Classes, interfaces and traits

Client¶

Memcached client class.

Instances of this class represents a connection to a memcached/yrmcds server. Most methods may throw \yrmcds\Error when a non-recoverable error happens.
« More »

Error¶

Exception for \yrmcds\Client.

Instances bring a meaningful message and an error code.
« More »

Response¶

Response from memcached / yrmcds server.

This class represents a command response from the server. All instances have these properties: * $status * $serial * $length * $command Other properties may or may not be available. Test their availabilities by `property_exists()`.
« More »

TimeoutException¶

This exception will be thrown if \yrmcds\SessionHandler cannot acquire the session lock.

« More »

SessionHandler¶

A session handler class based on \yrmcds\Client .

This class requires PHP5.4+ because this implements \SessionHandlerInterface available since PHP5.4. This class only works with yrmcds because server-side locking is not implemented in memcached. To set the session handler, pass an instace of this class to `session_set_save_handler()`.
